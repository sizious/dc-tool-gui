17. FAQ

Cette section traite des questions fréquemment posées (frequently asked questions).

1) Quand je lance une opération, j'obtiens le message CYGWIN1.DLL (ou CYGINTL.DLL) absent.
Il s'agit soit d'une erreur de votre part, soit d'une erreur de ma part. Vous pouvez configurer vos DLL dans la boite de dialogue Config > Configurer les DLL Cygwin... . Si vous avez coché dans cette boîte Utiliser les DLL internes, c'est qu'il y'a un problème. Soit une autre application a effacé les DLL Cygwin dans le dossier Temp de Windows, soit c'est une erreur très bizarre...

2) Lorsque j'upload un fichier, la barre se vide, se remplit, se vide, puis se remplit d'un coup.
Cette 'erreur' n'est pas vraiment une erreur. Vous envoyez un fichier ELF. Or ces fichiers ne s'uploade pas d'un coup, mais de section en section. Elle se remplit pour la première section, se vide puis se remplit avec la deuxième section...

3) J'utilisais le programme normalement, lorsque j'ai eu une Access Violation, avec une question.
Je suis désolé. Cette erreur n'est pas 'standard'. Elle peut arriver n'importe quand. Je travaille pour savoir d'ou elles peuvent venir (car elles ont plusieurs causes). Cliquez sur Oui pour fermer la boîte de dialogue et DC-TOOL GUI. Non pour tenter de travailler.

4) La barre de progression ne se remplit pas.
Soit le fichier est trop petit, soit il y'a eu un problème de synchronisation avec DC-TOOL. Désolé pour ce problème.

5) Quand je quitte l'application, DC-TOOL GUI me demande si je suis sûr(e) alors qu'il n'y a rien de spécial.
La encore, j'ai eu quelques problème à ce niveau là : DC-TOOL GUI a perdu la synchronisation avec DC-TOOL...

6) DC-TOOL GUI me dit "C:\DO[]U[]E~1\<user>\LOCALS~1\TEMP\DC-TOOL.EXE" introuvable.
Voilà autre chose... Une erreur dite "à la c*n". Impossible de savoir d'ou ca vient. Désolé.

7) Impossible de réduire la fenêtre de progression !
 Encore une erreur... pas terrible. Tentez de re-initialiser DC-TOOL GUI avec la commande Fichier > Re-Initialiser. Si ca ne marche toujours pas, vous devrez fermer DC-TOOL GUI... désolé.

8) Lorsque j'ai lancé DC-TOOL GUI alors que j'avais une opération avec DC-TOOL, DC-TOOL s'est fait fermer!
 C'est normal. DC-TOOL GUI ferme à chaque fois DC-TOOL. De toute façon si vous utilisez le GUI, vous n'utilisez plus le DC-TOOL normal!

9) J'utilisais normalement DC-TOOL GUI, quand j'obtiens un message d'erreur qui m'empêche de lancer DC-TOOL.
 Une autre application a effacé les fichiers nécessaires à DC-TOOL GUI. Les fichiers sont :

  • CYGWIN1.DLL
  • CYGINTL.DLL
  • DC-TOOL.EXE
  • DC-TOOL-IP.EXE

Ils sont situés dans le dossier temporaire de Windows (C:\Windows\Temp pour 9x/Me et C:\Documents and Settings\<username>\Local Settings\Temp pour NT/2K/XP).

10) A quoi sert la DLL DCTOOL.DLL ?
 Bah c'est une DLL de ressources que j'ai faite pour éviter que mon DC-TOOL GUI fasse 5 Mb. Je prefere séparer DC-TOOL GUI des programmes interne DC-TOOL, DC-TOOL-IP, de Cygwin...

11) J'utilisais normalement le programme, lorsque je l'ai quitté, il plante.
 Et oui, encore un problème de synchronisation. Lorsque vous quittez DC-TOOL GUI, il tue tous les process DC-TOOL & DC-TOOL-IP ce qui peut entrainer un plantage. Je travaille sur le sujet pour essayer de corriger ce problème.

12) Argh! Le PC a freeze (=gelé, bloqué) quand j'ai utilisé DC-TOOL GUI !
 Encore un autre problème difficile à résoudre... Je pense que c'est DC-TOOL lui même qui fait planter. J'espère que ca va pas vous le faire... :(

D'autres questions viendront. DC-TOOL GUI étant un jeune programme, il faut lui laisser le temps...
